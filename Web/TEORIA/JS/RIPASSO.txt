#########
#       #
#  DOM  #
#       #
#########

Il DOM (Document Object Model) è una rappresentazione strutturata del documento HTML sotto forma di oggetti JavaScript. 
Ogni elemento HTML diventa un nodo del DOM, organizzato in una struttura ad albero con relazioni padre‑figlio.

Il DOM funge da collegamento diretto tra il codice JavaScript e l’interfaccia utente visualizzata nel browser. 
Attraverso il DOM è possibile intercettare le interazioni dell’utente (click, input, submit, ecc.) e modificare dinamicamente la pagina.

Utilizzando JavaScript è possibile:

* selezionare elementi HTML
* modificarne il contenuto
* cambiarne lo stile
* aggiungere o rimuovere elementi

La manipolazione diretta del DOM è potente ma può diventare complessa e poco performante in applicazioni grandi, 
motivo per cui librerie come React gestiscono il DOM in modo astratto.

################
#              #
#  JAVASCRIPT  #
#              #
################

JavaScript è un linguaggio di programmazione interpretato, dinamico e orientato agli oggetti, nato inizialmente per rendere le pagine web interattive.

Oggi JavaScript non è più limitato ai browser: grazie a tecnologie come "Node.js" e "Deno" può essere eseguito anche lato server. 
Inoltre, con strumenti come "React Native" o "Capacitor", è possibile sviluppare applicazioni mobile e desktop.

Per utilizzare JavaScript in una pagina web esistono due approcci principali:

* inserire il codice direttamente in HTML tramite il tag `<script>`
* importare file JavaScript esterni

#####################
#                   #
#  IMPORT E EXPORT  #
#                   #
#####################

I moduli JavaScript permettono di suddividere il codice in file riutilizzabili.

**Export default** consente di esportare un singolo valore:

```
export default "chiave_secreta";
```

Importazione:

```
import miaChiave from './util.js';
```

**Export multipli** permettono di esportare più elementi:

```
export let API_KEY = "chiave_secreta";
export let ANOTHER_VARIABLE = "altro_valore";
```

Importazione:

```
import { API_KEY, ANOTHER_VARIABLE } from './util.js';
```

È possibile importare tutto in un unico oggetto:

```
import * as util from './util.js';
```

###############
#             #
#  VARIABILI  #
#             #
###############

Le variabili in JavaScript permettono di memorizzare dati come stringhe, numeri, booleani e valori speciali come `null` e `undefined`.

* `let` → variabile riassegnabile con scope di blocco
* `const` → valore costante non riassegnabile
* `var` → scope di funzione (sconsigliato)

Esempio:

```
let nome = "Mario";
nome = "Luigi";

const eta = 30;
```

Il comportamento di **hoisting** riguarda principalmente `var`, rendendolo fonte di bug. Per questo motivo si preferisce l’uso di `let` e `const`.

####################
#                  #
#  ARROW FUNCTION  #
#                  #
####################

Le Arrow Function sono una sintassi moderna e compatta per definire funzioni. Non possiedono un proprio `this`, ma lo ereditano dal contesto in cui sono definite.

```
const saluta = (nome) => {
    console.log("Ciao " + nome);
};
```

Sono ideali come callback e negli eventi React.

#############
#           #
#  OGGETTI  #
#           #
#############

Gli oggetti permettono di raggruppare dati correlati sotto un’unica struttura.

```
const utente = {
    nome: 'Rob',
    eta: 46,
    saluta() {
        console.log(this.nome);
    }
};
```

Gli oggetti possono essere creati anche tramite classi, che fungono da modello.

###########
#         #
#  ARRAY  #
#         #
###########

Gli array rappresentano collezioni ordinate di valori.

```
const hobbies = ['sport', 'cucina', 'lettura'];
```

Metodi comuni:

* `push()` → aggiunge elementi
* `findIndex()` → trova un indice
* `map()` → trasforma gli elementi

Gli array e gli oggetti sono assegnati **per riferimento**.

######################
#                    #
#  DESTRUTTURAZIONE  #
#                    #
######################

La destrutturazione permette di estrarre valori da array e oggetti in modo conciso.

```
const [nome, cognome] = ['Rob', 'Del'];

const { name, age } = user;
```

È molto usata nei parametri delle funzioni e in React.

#####################
#                   #
#  SPREAD OPERATOR  #
#                   #
#####################

Lo spread operator (`...`) consente di copiare o unire array e oggetti.

```
const nuovoArray = [...array1, ...array2];
const nuovoOggetto = { ...obj1, ...obj2 };
```

#############################
#                           #
#  FUNZIONI COME PARAMETRI  #
#                           #
#############################

In JavaScript le funzioni possono essere passate come argomenti ad altre funzioni.

```
setTimeout(() => {
    console.log('Eseguito dopo 3 secondi');
}, 3000);
```

Questo concetto è alla base delle callback e della programmazione asincrona.

#############
#           #
#  PROMISE  #
#           #
#############

Le Promise rappresentano operazioni asincrone che possono completarsi in futuro.

Stati:

* Pending
* Fulfilled
* Rejected

  miaPromise
  .then(risultato => console.log(risultato))
  .catch(errore => console.log(errore));

#################
#               #
#  ASYNC/AWAIT  #
#               #
#################

`async` e `await` permettono di scrivere codice asincrono in modo più leggibile.

```
async function caricaDati() {
    try {
        const risultato = await miaPromise;
        console.log(risultato);
    } catch (errore) {
        console.log(errore);
    }
}
```
