###############
#             #
#  VARIABILI  #
#             #
###############

Le variabili possono essere immaginate come delle **scatole** o contenitori all’interno dei quali memorizzare dei dati da riutilizzare nel tempo. Ogni variabile ha un nome (identificatore) e un valore associato.

In JavaScript una variabile può essere creata semplicemente assegnandole un valore. In passato era comune utilizzare la keyword `var`, anche se oggi si preferiscono `let` e `const`.

Quando si assegna un nome a una variabile è importante rispettare alcune regole:

* non possono esserci spazi
* si utilizza la convenzione **camelCase**
* può iniziare con `_`, `$` oppure una lettera

JavaScript consente la creazione implicita delle variabili, ovvero senza dichiarazione esplicita:

```
variabile = 3;
```

Questa pratica è fortemente sconsigliata perché la variabile diventa globale e accessibile da qualsiasi punto dello script, 
aumentando il rischio di errori e conflitti.

È quindi buona norma dichiarare sempre le variabili:

```
var miaVariabile = 3;
```

############################################
#                                          #
#  COMMENTI, PUNTO E VIRGOLA, E MAIUSCOLE  #
#                                          #
############################################

Il codice JavaScript è costituito da una sequenza di istruzioni eseguite dal motore JavaScript. Ogni istruzione può essere terminata da un punto e virgola.

```
var x = 5;
x = x + 1;
```

In JavaScript il punto e virgola **non è obbligatorio**, poiché il parser applica automaticamente l’Automatic Semicolon Insertion (ASI). 
Tuttavia, per chiarezza e stile, è consigliato usarlo.

JavaScript supporta due tipi di commenti:

* commenti su singola riga `//`
* commenti multi‑riga `/* */`

Tutto ciò che è commentato viene ignorato dal motore JavaScript.
È importante ricordare che JavaScript è **case sensitive**, quindi distingue tra lettere maiuscole e minuscole.

##############
#            #
#  STRINGHE  #
#            #
##############

Le stringhe rappresentano sequenze di caratteri e possono essere delimitate da apici singoli o doppi.

```
"stringa uno"
'stringa due'
```

Il delimitatore iniziale deve corrispondere a quello finale. Questo permette di inserire apici all’interno della stringa senza errori.
Esiste anche la **stringa vuota**, ovvero una stringa senza caratteri:

""
''

############################
#                          #
#  ESCAPE DI UN CARATTERE  #
#                          #
############################

Per utilizzare caratteri speciali all’interno delle stringhe si utilizza il backslash `\`.

```
console.log("questo è un \"test speciale\"");
```

########################
#                      #
#  CARATTERI SPECIALI  #
#                      #
########################

JavaScript supporta diversi caratteri speciali utili per la formattazione del testo:

```
- \n  nuova riga
- \t  tabulazione
- \r  ritorno a capo
- \\  backslash
```

################################
#                              #
#  CONCATENAZIONE DI STRINGHE  #
#                              #
################################

Le stringhe possono essere concatenate tramite l’operatore `+`:

```
"piano" + "forte"  // "pianoforte"
```

Le stringhe sono oggetti e dispongono di proprietà e metodi.

Proprietà:

* `length`

Metodi comuni:

* `toUpperCase()`
* `toLowerCase()`
* `split()`
* `replace()`

############
#          #
#  NUMERI  #
#          #
############

JavaScript utilizza un unico tipo numerico basato sulla virgola mobile a 64 bit. Non esiste distinzione formale tra numeri interi e decimali.

```
var intero = 10;
var decimale = 3.14;
```

Funzioni utili:

* `toFixed()`
* `parseInt()`
* `parseFloat()`

#########
#       #
#  NaN  #
#       #
#########

`NaN` significa *Not a Number* e indica un risultato numerico non valido.

```
NaN !== NaN
```

Per verificare un valore NaN si utilizza:

```
isNaN(valore);
```

################################
#                              #
#  NULL, UNDEFINED E BOOLEANO  #
#                              #
################################

* `null` rappresenta un valore intenzionalmente vuoto
* `undefined` indica un valore non assegnato
* `boolean` può assumere solo `true` o `false`

#################
#               #
# TIPIZZAZIONE  #
#               #
#################

JavaScript è un linguaggio a **tipizzazione debole e dinamica**: il tipo di una variabile dipende dal valore assegnato.

```
var test = "ciao";
test = 1;
```

############
#          #
#  TYPEOF  #
#          #
############

L’operatore `typeof` permette di conoscere il tipo di una variabile:

```
var test = "hello";
typeof test; // string

var x;
typeof x; // undefined
```
